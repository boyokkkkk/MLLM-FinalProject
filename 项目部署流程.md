# 项目部署流程（团队统一SOP）

更新时间：2026-05-28

## 1. 首次部署步骤

### 1.1 拉取代码

```bash
git clone <repo_url>
cd MLLMProject
```

### 1.2 创建虚拟环境（项目根目录）

```bash
python -m venv .venv
```

激活：

```bash
.\.venv\Scripts\Activate.ps1
```

### 1.3 安装依赖

```bash
pip install -r requirements.txt
```

### 1.4 配置环境变量

在项目根目录创建 `.env`：
目前我使用的是百炼API，模型和url我后续更新一般到文档里，如果需要切换本地模型，可群里商量。
```env
OPENAI_API_KEY=your_key
DASHSCOPE_API_KEY=your_dashscope_key
VLM_BASE_URL=...
```

## 2. 数据集部署

### 2.1 下载数据（推荐官方层+项目层）

```bash
python scripts/01_download_datasets.py --from-hf --hf-official-layout
```

产物说明：

1. `data/raw_hf/`：HF 官方结构（Arrow，完整数据）后续嫌占地方就删
2. `data/raw/`：项目交换层（json 导出）

### 2.2 校验与标准化（当前推荐 eval 模式因为docvqa没有train）

```bash
python scripts/01_prepare_datasets.py validate --mode eval
python scripts/01_prepare_datasets.py prepare --mode eval
```

产物说明：

1. `data/processed/docvqa/*.jsonl`
2. `data/processed/chartqa/*.jsonl`


### 2.3 离线文档切分与索引

快速烟测一两个样本：

```bash
python scripts/10_run_data_index_pipeline.py --skip-download --prepare-from-hf-cache --limit-per-split 2 --run-mllm-smoke --mllm-dry-run --mllm-num-samples 2
```

完整使用当前 eval 标准化产物构建索引：

```bash
python scripts/07_parse_and_chunk.py --datasets docvqa,chartqa --splits val,test
python scripts/08_build_indexes.py
python scripts/09_query_indexes.py "actual value per 1000 during 1975" --top-k 3
```

产物说明见 `docs/data_indexing.md`，索引清单输出到 `data/processed/indexes/index_manifest.json`。

## 3. 服务启动

### 3.1 启动后端 API
没有办法用模块运行就正常运行scripts下的代码就行
```bash
python -m src api
```

### 3.2 启动前端 UI（Streamlit）

```bash
python -m src ui
```

## 4. 团队协作

1. 仅在项目 `.venv` 中运行命令，禁止污染系统 Python。
2. 大体积数据不提交 Git（`data/raw*`, `data/processed` 已在 `.gitignore`）。
3. 数据产物只通过脚本生成，不手改 `data/processed/*.jsonl`。
4. 合并前最少执行一次：

```bash
python scripts/01_prepare_datasets.py validate --mode eval
```

## 5. 常见问题

### 5.1 `ModuleNotFoundError: No module named 'src'`

处理方式：

1. 确认在项目根目录执行命令
2. 确认已执行 `pip install -e .`
3. 优先用 `python -m src api` / `python -m src ui`
4. 或者直接改下报错文件的import

### 5.2 数据校验提示缺少 `docvqa/train.json`

说明：当前默认 DocVQA 源不含 train split。
请使用：

```bash
python scripts/01_prepare_datasets.py validate --mode eval
python scripts/01_prepare_datasets.py prepare --mode eval
```
