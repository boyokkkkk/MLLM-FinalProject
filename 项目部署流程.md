# 项目部署流程（团队统一SOP）

更新时间：2026-05-28

## 1. 首次部署步骤

### 1.1 拉取代码

```bash
git clone <repo_url>
cd MLLMProject
```

### 1.2 创建虚拟环境（项目根目录）

```bash
python -m venv .venv
```

激活：

```bash
.\.venv\Scripts\Activate.ps1
```

### 1.3 安装依赖

```bash
pip install -r requirements.txt
```

### 1.4 配置环境变量

在项目根目录创建 `.env`：
目前我使用的是百炼API，模型和url我后续更新一般到文档里，如果需要切换本地模型，可群里商量。
```env
OPENAI_API_KEY=your_key
DASHSCOPE_API_KEY=your_dashscope_key
VLM_BASE_URL=...
```

## 2. 数据集部署

### 2.1 下载数据（推荐官方层+项目层）

```bash
python scripts/01_download_datasets.py --from-hf --hf-official-layout
```

产物说明：

1. `data/raw_hf/`：HF 官方结构（Arrow，完整数据）后续嫌占地方就删
2. `data/raw/`：项目交换层（json 导出）

### 2.2 校验与标准化（当前推荐 eval 模式因为docvqa没有train）

```bash
python scripts/01_prepare_datasets.py validate --mode eval
python scripts/01_prepare_datasets.py prepare --mode eval
```

产物说明：

1. `data/processed/docvqa/*.jsonl`
2. `data/processed/chartqa/*.jsonl`


### 2.3 离线文档切分与索引

当前 RAG 设计采用 block-first schema：MinerU 输出的 `text/table/figure/formula` 等 block 才是默认 embedding 和索引单元；DocVQA / ChartQA 的 `question` 是用户 query，`answers` 是评测标签，默认不进入数据库。

快速烟测一两个评测样本：

```bash
python scripts/10_run_data_index_pipeline.py --skip-download --prepare-from-hf-cache --limit-per-split 2 --run-mllm-smoke --mllm-dry-run --mllm-num-samples 2
```

使用已有 DocVQA / ChartQA 标准化产物和 MinerU blocks 构建索引：

```bash
python scripts/07_parse_and_chunk.py --source-mode benchmark --datasets docvqa,chartqa --splits val,test
python scripts/08_build_indexes.py
python scripts/09_query_indexes.py "actual value per 1000 during 1975" --top-k 3
```

如果只是早期调试，确实想把评测问题/答案也生成 synthetic chunk，需要显式加 `--include-qa-context`；正常 RAG 不加：

```bash
python scripts/07_parse_and_chunk.py --source-mode benchmark --include-qa-context --datasets docvqa --splits val --limit-per-split 1
```

产物说明见 `docs/data_indexing.md`，索引清单输出到 `data/processed/indexes/index_manifest.json`。


### 2.4 DocVQA 数据流和字段说明

DocVQA 本地数据按以下顺序流动：

```text
data/raw_hf/docvqa/                    # HF Arrow 官方结构，保留真实图片列
  -> data/raw/docvqa/{val,test}.json    # 项目交换层 JSON，便于查看字段
  -> data/images/docvqa/<split>/*.png   # 从 HF Arrow 解码出的本地图片
  -> data/processed/docvqa/*.jsonl      # 统一 QA 样本，question 是 query
  -> data/interim/mineru/docvqa/*.json  # MinerU 归一化 blocks
  -> data/processed/documents/documents.jsonl
  -> data/processed/chunks/chunks.jsonl # block 级 embedding / retrieval 单元
  -> data/processed/indexes/*
```

推荐 DocVQA 小样本真实链路：

```bash
python scripts/00_download_mineru_models.py
python scripts/10_run_data_index_pipeline.py --skip-download --prepare-from-hf-cache --datasets docvqa --splits val --limit-per-split 1 --run-mineru --mineru-model-source modelscope --run-mllm-smoke --mllm-dry-run --mllm-num-samples 1
```

关键中间格式：

1. `data/processed/docvqa/val.jsonl`

```json
{"id":"49153","dataset":"docvqa","split":"val","question":"...","answers":["0.28"],"image":"data/images/docvqa/val/49153.png","metadata":{"questionId":"49153","docId":14465,"ucsf_document_page_no":"81"}}
```

这里 `image` 对应真实系统里的输入文档页面，`question` 对应用户提问，`answers` 只用于评测。

2. `data/interim/mineru/docvqa/49153.json`

```json
{"sample_id":"49153","blocks":[{"block_id":"block-0000","type":"figure","text":"figure region","bbox":[118,226,870,858],"page_no":1}]}
```

3. `data/processed/documents/documents.jsonl`

```json
{"document_id":"doc-...","sample_id":"49153","source_path":"data/images/docvqa/val/49153.png","image_path":"data/images/docvqa/val/49153.png","page_no":81,"question":"...","answers":["0.28"]}
```

4. `data/processed/chunks/chunks.jsonl`

```json
{"chunk_id":"chk-...","document_id":"doc-...","block_id":"block-0000","chunk_type":"figure","text":"figure region","page_no":1,"bbox":[118,226,870,858],"source_ref":"docvqa/val/49153#page=1#figure=fig-49153#block=block-0000","source_path":"data/images/docvqa/val/49153.png","image_path":"data/images/docvqa/val/49153.png"}
```

`chunks.jsonl` 是后续向量库真正要保存 embedding 的对象。检索命中 `chunk_id` 后，靠 `document_id/block_id/page_no/bbox/source_path/image_path` 回到原文档和页面区域。

5. `data/processed/indexes/`

- `text/doc_store.json`: chunk 全量信息、词频和回溯字段。
- `text/postings.json`: token 到 chunk 的倒排表。
- `text/document_frequency.json`: 每个 token 出现于多少 chunk。
- `vision/visual_store.json`: figure/table/formula/page_image 等视觉块元数据。
- `index_manifest.json`: 索引版本、构建时间、输入输出路径和数量。

脚本职责：

- `01_download_datasets.py`: 下载 HF 数据并导出 `data/raw`。
- `01_prepare_datasets.py`: 标准化 QA 字段，并从 `data/raw_hf` 导出真实图片。
- `00_download_mineru_models.py`: 从 ModelScope 下载 MinerU 权重。
- `06_run_mineru.py`: 调用 MinerU，输出归一化 blocks；也支持 `--input-path` 直接处理普通 PDF/图片。
- `07_parse_and_chunk.py`: 生成 document/chunk schema，默认只把 MinerU block 写成 chunk。
- `08_build_indexes.py`: 构建文本索引和视觉元数据索引。
- `09_query_indexes.py`: 查询索引并输出可回溯引用。
- `10_run_data_index_pipeline.py`: 串起 prepare、MinerU、chunk、index、MLLM smoke。

更多字段解释见 `docs/data_indexing.md`。


### 2.5 普通 PDF 输入烟测

用户上传 PDF 的场景可用 `data/raw/2511.05491v1.pdf` 模拟。先跑第 1 页，确认 MinerU、切分和索引字段链路：

```bash
python scripts/06_run_mineru.py \
  --input-path data/raw/2511.05491v1.pdf \
  --datasets raw_pdf \
  --splits raw \
  --limit-per-split 1 \
  --backend pipeline \
  --mineru-model-source modelscope \
  --start-page 0 \
  --end-page 0

python scripts/07_parse_and_chunk.py --source-mode mineru --datasets raw_pdf --splits raw --limit-per-split 1
python scripts/08_build_indexes.py
```

已验证结果：

```text
MinerU normalized blocks: 12
documents.jsonl: 1
chunks.jsonl: 14
index text_terms: 288
index visual_items: 0
```

`visual_items=0` 是因为该 PDF 第 1 页识别结果都是文本块；有图、表、公式的页面会进入视觉元数据索引。

## 3. 服务启动

### 3.1 启动后端 API
没有办法用模块运行就正常运行scripts下的代码就行
```bash
python -m src api
```

### 3.2 启动前端 UI（Streamlit）

```bash
python -m src ui
```

## 4. 团队协作

1. 仅在项目 `.venv` 中运行命令，禁止污染系统 Python。
2. 大体积数据不提交 Git（`data/raw*`, `data/processed` 已在 `.gitignore`）。
3. 数据产物只通过脚本生成，不手改 `data/processed/*.jsonl`。
4. 合并前最少执行一次：

```bash
python scripts/01_prepare_datasets.py validate --mode eval
```

## 5. 常见问题

### 5.1 `ModuleNotFoundError: No module named 'src'`

处理方式：

1. 确认在项目根目录执行命令
2. 确认已执行 `pip install -e .`
3. 优先用 `python -m src api` / `python -m src ui`
4. 或者直接改下报错文件的import

### 5.2 数据校验提示缺少 `docvqa/train.json`

说明：当前默认 DocVQA 源不含 train split。
请使用：

```bash
python scripts/01_prepare_datasets.py validate --mode eval
python scripts/01_prepare_datasets.py prepare --mode eval
```
